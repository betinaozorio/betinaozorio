# QA Engineer

**7+ years · Brazil**

Before I write a single test case, I need to understand the system. Not just the feature — the whole system.

Early in my career I was handed a legacy COBOL order management system at a global tech company and told to test it. No specs. No guides. I mapped the system through conversations with developers, systematic observation, and exploratory sessions until I understood what it did and where it could fail. That instinct for investigation became my method.

Since then: e-commerce platforms with 1M+ active users, a loyalty program with 500k+ participants, iOS/Android mobile apps, REST APIs, and logistics systems for a Fortune 500 company. Same approach every time: understand the business, understand the flows, find where the assumptions break.

Now expanding into test automation with Playwright (TypeScript), with hands-on projects here on GitHub, and deepening my backend knowledge through a Java bootcamp.

Open to remote QA and SDET opportunities.
